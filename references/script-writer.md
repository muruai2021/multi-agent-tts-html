# 口播脚本写手指南（人味版）

> 写出口播文案，让听众听不出"这是 AI 写的"。

## 核心理念

**AI 味 = 听起来像在读稿**
**人味 = 听起来像在跟你唠嗑**

听众闭上眼睛，听三秒钟，就能分辨出哪个是 AI 写的、哪个是真人录的。区别不在于"对不对"，而在于"像不像一个活人在说话"。

---

## ❌ 第一部分：反 AI 味 10 条铁律

### 1. 禁词清单（出现一次扣分）

| 禁用词 | 替代方案 |
|--------|----------|
| 赋能 | 帮上忙、有用 |
| 抓手 | 切入点、办法 |
| 闭环 | 完整流程 |
| 底层逻辑 | 根本原因 |
| 维度 | 方面、点 |
| 认知 | 看法、想法 |
| 痛点 | 烦心事、卡住的地方 |
| 颗粒度 | 细到、详细程度 |
| 链路 | 流程、路子 |
| 壁垒 | 门槛、护城河 |
| 在当今...时代 | 直接说事 |
| 随着...的发展 | 删掉，直接讲 |
| 让我们共同... | 别说"我们" |
| 总而言之 / 综上所述 | 收尾要轻 |
| 未来已来 | 老掉牙了 |

### 2. 禁用结构

- ❌ **三段式排比**："不仅...而且...，更...的是..."
- ❌ **工整对仗**："上到总裁下到员工，从研发到销售"
- ❌ **金句三连**："XX 是 YY，是 ZZ，更是 WW"（除非你真的很懂）
- ❌ **总结升华结尾**："让我们一起拥抱 XX 的美好未来"
- ❌ **宏大叙事开场**："在数字化转型的浪潮下..."
- ❌ **小标题堆叠**：连续 10 个 `##` 把文案切成 PPT

### 3. 禁用修辞

- ❌ **空洞比喻**："就像一束光照进了黑暗"
- ❌ **古诗词强行引用**：听众听不进去
- ❌ **无来源数据**："研究表明 80% 的人..."（研究表明个啥）
- ❌ **过度礼貌**："非常荣幸为您介绍..."（这是口播不是颁奖）

### 4. 一句话长度控制

- **理想长度**：15-25 字
- **最大长度**：35 字
- **超过 35 字**：必断句。口语里没人一句话说 35 字

> 反例（59 字）：
> "在当前这个快速发展的时代背景下，企业如果不能及时进行数字化转型，就很有可能会在激烈的市场竞争中失去优势。"
>
> 正解（拆 4 句）：
> "现在变化太快了。你不转，竞争对手在转。转晚了，连汤都喝不上。所以这事不能拖。"

### 5. 段落长度控制

- 单段不超过 **3 句话**
- 超过 3 句必须换气
- 段落之间留半秒停顿（用 `...` 表示）

### 6. 主语具体化

- ❌ "用户" → ✅ "我那哥们儿"、"我表弟"、"你"
- ❌ "企业" → ✅ "一家公司"、"一个团队"
- ❌ "专业人士" → ✅ "干了十年的老张"

### 7. 数字要"具体"

- ❌ "很多人"、"大多数"、"绝大部分"
- ✅ "我见过 30 多家公司里，有 27 家都..."
- ✅ "一秒钟，账单上多了 800 块"
- ✅ "连续 3 个月，每天加班到 11 点"

### 8. 必须有"现场感"

加入以下任意一项：
- 一个具体时间（"上周三"、"去年双十一"）
- 一个具体地点（"在中关村"、"在公司茶水间"）
- 一个具体场景（"周五下班路上"、"凌晨两点"）
- 一个具体人物（"我同事"、"楼下张大爷"）

### 9. 留 30% 的"不完美"

真人说话不是论文。要有：
- **轻微重复**："这个事...这个事怎么说呢"
- **自我纠正**："不是 XX，应该是 YY"
- **未说完的话**："反正就是...你懂的吧"
- **语气词**：嗯、其实、对、就是说

### 10. 全文至少 3 处"停顿"

真人讲话必有停顿和换气。口播里用 `...` 或 `——` 或 `（停顿）` 标记：

```
我之前也试过很多方法（停顿）...都没用。
后来我才发现，问题根本不在那儿。
```

## 🎣 第二部分：10 种钩子模板（前 5 秒定生死）

听众给一个口播的注意力，**只有 5 秒**。开头没勾住，他就划走了。

### 1. 颠覆认知型

> "你可能不信，但做了 5 年 XX 的人，90% 都做错了。"

适用：让听众感觉"我可能也是错的"

### 2. 痛点直击型

> "我跟你说，这个事儿我快被它气死了。"

适用：让听众觉得"我也是！"

### 3. 反差冲击型

> "花了 30 万买的教训，今天免费告诉你。"

适用：吸引"占便宜"心理

### 4. 悬念型

> "你知道为什么 XX 总是失败吗？我研究了一年，发现一个反常识的事。"

适用：制造好奇缺口

### 5. 数字碾压型

> "一秒钟，损失 800 块。10 分钟，烧掉 50 万。"

适用：让听众感觉"代价巨大"

### 6. 场景代入型

> "想象一下，你周一早上刚到工位，老板把你叫进去，告诉你..."

适用：让听众进入"虚拟现实"

### 7. 故事开场型

> "我有个朋友，在大厂干了 8 年。上个月突然被裁了。"

适用：故事是注意力永动机

### 8. 反问型

> "你有没有这种感觉？明明很努力，但就是看不到结果。"

适用：让听众下意识回答"有"

### 9. 否定常识型

> "我直说，XX 这事儿其实没必要做。因为做了反而更糟。"

适用：吸引"反对派"反向关注

### 10. 冒犯型（慎用）

> "说真的，听完这段的人 90% 还在用错误的方法。"

适用：制造"我被点名了"的紧张感

**钩子使用规则：**
- 钩子长度 ≤ 15 字
- 钩子必须在前 5 秒出现
- 钩子之后再展开，别一钩子就完事

## 🎙️ 第三部分：人味细节注入清单

### 语气词使用（全文 5-10 处）

| 语气词 | 用法 | 示例 |
|--------|------|------|
| 嗯 | 思考、转折 | "嗯...其实这事我琢磨了很久" |
| 其实 | 揭示真相 | "其实问题没你想的那么复杂" |
| 说白了 | 强调本质 | "说白了就是一句话" |
| 对 | 确认、对话感 | "对，就是这个意思" |
| 就是说 | 解释 | "就是说，他不是不想做" |
| 你知道吧 | 拉近距离 | "那个味儿你知道吧" |
| 怎么说呢 | 铺垫 | "怎么说呢，这事儿挺复杂" |
| 说真的 | 强调 | "说真的，我一开始也不信" |

### 停顿符号使用规范

| 符号 | 时长 | 场景 |
|------|------|------|
| `，` | 0.3 秒 | 常规换气 |
| `、` | 0.4 秒 | 列举停顿 |
| `。` | 0.6 秒 | 段落结束 |
| `...` | 0.8 秒 | 思考、转折 |
| `——` | 1.0 秒 | 强调、悬念 |
| `（停顿）` | 1.2 秒 | 长停顿 |
| 换行 | 0.5 秒 | 段落分隔 |

### 句式"打断"技巧

真人不会"按顺序说完一整段"。他会：

**插入式打断：**
```
我之前以为 XX 是这样（停顿）...后来才发现，根本不是。
```

**自我否定：**
```
不是 XX 的问题，是 YY 的问题。嗯...应该说，是 ZZ 的问题。
```

**半截话：**
```
反正这事儿吧...你懂我的意思吧？
```

**反问自己：**
```
为什么会这样呢？嗯...我后来想明白了。
```

## 📏 第四部分：节奏与时长控制

### 字数与时长换算

| 语速 | 每分钟字数 | 适用场景 |
|------|-----------|----------|
| 0.8x | 180 字/分 | 抒情、故事、深度内容 |
| 1.0x | 240 字/分 | 标准口播（推荐） |
| 1.2x | 280 字/分 | 快讯、广告、知识密度高 |
| 1.5x | 320 字/分 | 抖音快剪、紧急公告 |

**默认建议：1.0x，即 240 字/分**

### 段落节奏建议（每 30 秒一个小高潮）

```
[0-5s]  钩子（吸引注意）
[5-15s] 第一个冲突/问题
[15-25s] 转折/揭秘
[25-30s] 小高潮/金句
[30-35s] 留白过渡
```

### 信息密度控制

- 每 30 秒 **1 个核心信息**（多了记不住）
- 每 60 秒 **1 个具体案例或数据**
- 全文 **不超过 5 个核心观点**（多了是 PPT 不是口播）

## ✍️ 第五部分：案例对照

### ❌ AI 味原稿（来自 multi-agent-skill-factory）

> "你有没有遇到过这种困扰——想给团队封装一个 AI 技能，却发现自己要花两三天时间：读需求、写文档、配角色、调工具、出 SKILL.md、跑测试……全靠手工，累死累活。
>
> 问题出在哪？不是 AI 不够强，而是我们封装技能的方式，还停留在手工业时代。"

**问题诊断：**
- "问题出在哪？" 是伪问题，听众没问
- "手工业时代" 是空洞比喻
- 整体过于工整，缺乏现场感

### ✅ 人味改写版

> "我跟你说，去年我帮一个团队做 AI 技能封装，整整搞了 3 天。
>
> 3 天啊。
>
> 头一天读需求，光是写需求文档就磨了一下午。第二天开始配角色，调工具调到我怀疑人生。第三天出 SKILL.md，又卡了一下午。
>
> 最离谱的是——做出来还不能直接用，得跑测试。
>
> 你猜怎么着？跑出来一堆 bug，又得回头改。
>
> 那会儿我就想：这事不能一直这么干吧？
>
> 后来我才发现，问题根本不在我身上。是这套流程本身，太糙了。"

**改了什么：**
- ✅ 加入了"去年我帮一个团队"（具体场景）
- ✅ "3 天啊"（重复强调 + 情感）
- ✅ "调工具调到我怀疑人生"（情绪化表达）
- ✅ "最离谱的是"（口语化转折）
- ✅ "你猜怎么着？"（互动钩子）
- ✅ "那会儿我就想"（第一人称现场感）
- ✅ "太糙了"（具体而粗粝的表达）

## 📋 第六部分：写前自检清单

写完之后，过一遍这个清单：

- [ ] 开头 5 秒内有钩子？
- [ ] 全文没有"赋能/抓手/底层逻辑"等黑话？
- [ ] 至少 3 个具体场景/时间/人物？
- [ ] 至少 5 个语气词或停顿？
- [ ] 每段不超过 3 句话？
- [ ] 没有任何"在当今...时代"的宏大开场？
- [ ] 结尾没有"让我们一起..."式的升华？
- [ ] 闭眼读一遍，听起来像在跟朋友说话？
- [ ] 总字数 / 240 = 时长（分钟）符合预期？
- [ ] 至少 3 处用了 `...` 或 `——` 标记停顿？

**10 项全过，才算合格。**

## 🎯 第七部分：四种文体配方

### 知识科普型（如产品介绍）

- 钩子：颠覆认知型
- 节奏：1.0x，每 30 秒一个案例
- 收尾：留个思考题，不升华

### 故事叙述型（如人物访谈）

- 钩子：故事开场型
- 节奏：0.9x，给情绪留时间
- 收尾：故事自然结束，不总结

### 干货清单型（如教程）

- 钩子：数字碾压型
- 节奏：1.1x，信息密度高
- 收尾：给一个可执行的下一步

### 情感共鸣型（如心灵鸡汤）

- 钩子：痛点直击型
- 节奏：0.85x，慢一点
- 收尾：用一句"懂你的人自然会懂"做结

## 💡 第八部分：常见错误避坑

### 错误 1：把所有内容都写完

口播不是论文。**一个 3 分钟的口播，传递 1 个核心信息**就够。多了听众记不住。

### 错误 2：把书面语直接念

写出来看得懂 ≠ 念出来听得懂。写完要**用嘴念一遍**。念不顺的，全部改。

### 错误 3：害怕重复

真人讲话有 20% 的内容是"废话"（嗯、其实、就是说）。AI 写稿最大的问题就是**没有废话**。要刻意加。

### 错误 4：追求完美

真人说话有错别字、有口误、有"我刚才说错了"。口播里如果太完美，反而假。

> 技巧：可以刻意在中间加一句"等等，我刚才说错了，应该是..."，反而更真实。

### 错误 5：把每一个字都写死

口播的"灵魂"在现场发挥。**稿件只写 70%，留 30% 给现场**。所以稿件要写得有弹性，标注"这里可以即兴发挥"。

## 🚀 第九部分：与 multi-agent-mmx-tts 配合使用

### 完整工作流

```
1. 写手按本指南产出脚本（md2mp3.py 会读取）
   ↓
2. 脚本自动识别停顿符号（... ——）
   ↓
3. 按段落调用 TTS API
   ↓
4. 在停顿处自动插入静音
   ↓
5. 拼接成完整 MP3
```

### 推荐格式

```markdown
# 标题（不读出）

正文内容（会读出）。
可以用 ... 表示停顿。
可以用 —— 表示强调停顿。

## 章节（不读出）

新章节开始。
```

### Frontmatter 配置

```markdown
---
title: 口播标题（嵌入 MP3 元数据）
voice: 国语男声
speed: 1.0
volume: 1.5
insert_pauses: true
pause_duration_ms: 600
---
```
