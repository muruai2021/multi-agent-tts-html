# 内容映射表模板（防虚构关键）

> **核心原则**：屏幕上显示的每一条文字，都必须在字幕/口播里有对应内容。真实感来自"对齐"，不来自"加戏"。

---

## 什么是内容映射表？

在排版之前，先列出**每个画面元素**与**字幕原文**的对应关系。

找不到字幕对应的元素，**一律删除或改写**——这是防虚构的核心关卡。

---

## 模板格式

| 画面元素 | 字幕对应 | 时段 | 状态 |
|---------|---------|------|------|
| 标题"..." | "..."（XX:XX 处字幕原话） | 0-10s | ✅ 对齐 |
| 数字 "200 万" | ❌ 字幕无此数字 | 15-25s | ⛔ 删除/改写 |
| 引言"王磊说..." | ❌ 字幕只有"一个朋友" | 30-40s | ⛔ 改为匿名 |
| 场景"北京海淀" | ❌ 字幕无地点 | 40-50s | ⛔ 删除地点 |

**状态说明**：

| 状态 | 含义 | 处理方式 |
|------|------|----------|
| ✅ 对齐 | 字幕原文有，页面有，完全对应 | 通过 |
| ⛔ 删除/改写 | 页面有，字幕无 | **必须删除**或**改写对齐字幕** |
| ⚠️ 部分对齐 | 字幕有类似内容但表达不同 | 改为字幕原文或删除 |

---

## 7 条戒律（每条都要填）

| # | 戒律 | 检查项 |
|---|------|--------|
| 1 | **不虚构人名** | 口播"一个朋友" → 排版禁止写"王磊 CTO" |
| 2 | **不虚构数字** | 口播无具体数字 → 排版禁止写"200 万/95%/3 周" |
| 3 | **不虚构场景** | 口播"5 种角色" → 排版禁止写"5 月 12 日 北京海淀" |
| 4 | **不虚构比喻** | 口播"串台" → 排版禁止写"10 个一模一样的抽屉" |
| 5 | **不虚构代码** | 口播"CLAUDE.md 写规矩" → 排版禁止写 12 行业务代码 |
| 6 | **不虚构流程** | 口播无流程 → 排版禁止写 D1-D7 七步 30 分钟 |
| 7 | **不虚构采访** | 口播无采访 → 排版禁止写"李婷/高远/周明 三段引言" |

## 真实感 4 个来源（只能来自字幕）

| 类型 | 来源 | 例子 |
|------|------|------|
| **真实数据** | 口播提到的具体数字 | "50 城 / 6 个 / 1+2+3 / 3 秒" |
| **真实人物** | 口播中提到的角色（可匿名） | "一个朋友" / "政策雷达" / "百事通" |
| **真实场景** | 口播描述的具体情境 | "上礼拜" / "凌晨三点" / "6 人小组" |
| **真实引言** | 口播原话或近义改写 | "三不知" / "长得一样、切得零成本" |

## 填写步骤

### Step 1：收集 3 个源
- [ ] 字幕源（Phase 0.3 输出的 `.srt` / `.vtt` / `captions.json`）
- [ ] 口播稿（Phase 0.1 输出的 `口播稿.md`）
- [ ] 时间戳映射表（字幕文件里自带）

### Step 2：逐画面标注

对 HTML 的**每一页/每一个元素**，在映射表里查找：
1. 这个元素的内容在字幕里能找到吗？
2. 如果能找到，是原文还是改写（允许近义改写）？
3. 如果找不到，标记 ⛔

### Step 3：处理 ⛔ 项

| 情况 | 处理 |
|------|------|
| 虚构人名 | 改为"一个朋友" / "某位观众"等匿名表述 |
| 虚构数字 | 删除数字，或改为字幕中真实出现的数字 |
| 虚构场景 | 删除场景描述 |
| 虚构比喻 | 删除比喻，保留字幕中的原话 |
| 虚构代码 | 删除代码块，只保留字幕中提及的概念名 |
| 虚构流程 | 改为口播中的实际步骤数 |
| 虚构采访 | 删除引言，改为字幕中的转述 |

### Step 4：确认 100% 覆盖

所有画面元素状态为 ✅ 对齐，无 ⛔ 项，方可进入 Phase 3 排版。

```markdown
## 口播稿自检（AI 内部）

- [ ] 具体角色/场景 ≥ 3 个
- [ ] 字数在 800-1200 之间
- [ ] 结尾有"评论区/你怎么看"等互动
```

## 与 GATE 2 的关系

Phase 3 排版完成后，GATE 2 审核会检查：

1. **7 条戒律核查** — 内容映射表的 ⛔ 项全部清零
2. **内容映射表覆盖** — 所有画面元素都标注 ✅ 对齐
3. **字幕同步预览** — 抽 5 帧验证字幕文本 = 字幕源

映射表 100% 覆盖（无 ⛔ 状态）是 GATE 2 通过的必要条件。
